## 1. 문제 / 목표
현재 입력은 클릭이 많고, "통과" 버튼을 누르면 즉시 통과 처리되어 기록이 사라진다. 강사가 수업 중간 진행을 남길 수 없다.

목표: ① 강사 1명 기록 10초 이내 ② 수업 중 실시간 기록 ③ 누적 데이터 = AI 피드백 기초 ④ **통과 중심 → 오늘 기록(성장) 중심**.

## 2. 데이터 원리 (스키마 변경 0)
| 동작 | 저장 |
|---|---|
| 칩 탭 = "오늘 했음" | `measurements(metric_type='attempt', skill_step_id, value=1)` — 토글 |
| 측정 입력(초/회/바퀴) | `measurements('time_sec'/'stroke_count'/'laps')` |
| 완주(통과) | `skill_progress` (append-only, unique) |
| 결석 | `sessions.attendance='결석'` upsert |

"오늘 기록" = 오늘자 measurements + 오늘자 skill_progress 합. 통과는 **선택적 부가 동작** → 탭해도 칩/카드 안 사라짐. 모든 행 append-only로 누적되어 성장 곡선·AI 리포트 재료가 됨.

## 3. 오늘 수업 화면 (`/v2/today`)
- 상단 "오늘 미입력 N명" 카운터.
- **수업 시간별 그룹**(예 "4시 수업 (12명)"), 현재 시간대 그룹 우선. (`lib/schedule` hour 사용)
- 학생 카드(큰 카드):
  - 헤더: 이름 · 반(수4시) · **영법 색 배지** · 최근 통과 trail · `진도→`
  - 칩 그리드: **현재 영법의 현재 칸 중심 창**(미통과 + 오늘통과, ladder_order 순, 최대 ~8). 칩 탭=오늘 기록(✓ 초록), 재탭=취소.
  - 측정 스텝: 탭 → 인라인 입력(초/회) + 저장 (`체크→입력→저장`).
  - 완주: 기록된 칩에 `완주` 핀 1탭 → `skill_progress`. **통과해도 수업 끝까지 회색 ✓로 잔류.**
  - 하단 "오늘 기록" 요약 + `결석` 코너 버튼.
- 카드 색: 기록≥1 → 초록(좌측 보더+옅은 바탕+"✓기록") / 결석 → 빨강 / 미입력 → 흰색.
- 정렬: 미입력 위, 기록완료 아래.

## 4. 출결 단순화
`출석`·`지각` 삭제. `결석`만(토글, 사유 빠른선택 옵션). 출석은 기록 시 암묵 처리(`ensureSession`).

## 5. 영법 색상 체계
초보=흰 / 자유형=노랑 / 배영=초록 / 평영=파랑 / 접영=보라 / 마스터=검정. `strokes.color` 사용(시드 점검·보정). 배지 색 vs 카드 초록은 라벨/보더로 구분.

## 6. 메뉴 / 라우팅
레이아웃 탭: **오늘 수업** | **나의 학생**.
- `/v2/today`
- `/v2/students` — 나의 학생 목록(신규)
- `/v2/student/[id]` — **대시보드**(기본정보·진도%·최근30일)
- `/v2/student/[id]/progress` — 전체 사다리 편집(현 `student/[id]` 이동, "진도→" 타깃, 통과 후에도 측정 추가)
- `/v2/student/[id]/baseline` — 유지

## 7. 나의 학생 대시보드
기본정보(이름·반·현재단계·담당) · 영법별 진도%(통과 ladder/전체 ladder) · 최근 30일 기록(measurements+progress 날짜순). 성장그래프·AI는 다음 단계(자리만).
